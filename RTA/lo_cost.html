1: What are the Machine-Code Emulators?

RTA1 is a project aimed at low-power semiconductor, a freeware programmable processor architecture introduced in emulation on PCs

New machine-code emulators execute RTA1 instructions in a tight loop of platform machine-code

The result is double the performance of earlier RTA1 emulators and half the demand on platforms

These faster and cooler emulators are initially delivered for Intel Linux 32-Bit and Intel OSX 32-Bit platforms

RTA1 is about getting serious performance from ultra-low power

It's an executable architecture with extraordinary software efficiency

RTA1 issues relatively few instructions to get work done

The instruction set is simple but high-geared. So much so that TCP / UDP / IP are implemented in 4000 instructions

At the same time, RTA1 has the largest floating-point number capability in the world.
When RTA1 emulators say floating point, they're not talking about what the platform has.
RTA1 has giant numeric capability unknown to workstations or even to prestige national computers

Emulated implementations run networked applications in continuous server mode and only ever stop for software upgrades or faster emulators in years of running

Downloads are free acquisition of a freeware product encompassing emulated machine, target software, and PC-based development and client softare

The aim of RTA1 is large capability for very little energy. Like when you leave stuff running off a watch battery for a year or more

What are these new emulators for? Well the old ones are fast enough for emulation, but they're cycle-guzzlers written in gcc

These new emulations are written in machine code and are closer to the RTA1 project's performance objectives for future semiconductor RTA1s

And certainly more suitable for a project about low power

The new emulators are twice as fast and the platforms work half as much

2: Where are the New Emulators and how do you have Them?

These new emulators are 32-bit Intel for Linux and Mac

One shell has rta/linux.net/runL or rta/portal/runL running

Another shell for emulator should be in directory rta/tgen

In rta/tgen you do ./ra instead of ./r

You need a third shell in rta/client to start routing to RTA1 with script ./prun

That routes to the RTA1 you just launched and any more RTA1s in your network

For more detail see Starting the Emulator

3: RTA1 Emulators Delivered by Platform

3.1: Things you need

Note that you can't run any RTA1 without masmx 7r3A which does many just-in-time assemblies, and you can't run any RTA1 without some binary utilities

3.2: 32-Bit and Clock-Wraparound-Year

Note that 32-bit emulators have a fix for future clock-wraparound, but this will only work if the clock is allowed to wrap around

Some platforms police the clock to jam at the start of wraparound-year

3.3: Delivered Emulators

Note 64-bit Linux platforms may need a 32-bit runtime download 

	The numbers on the end of emulator names are
	how many 262144-word blocks of excutable space an emulator has
	zero means half a block or 131072 24-bit words

	bash loads the emulator of size corresponding to $RTA_MBANKS

	all emulators pass the same benchmarks

	the very largest ones use a lot of PC virtual space, but don't use very much of it at once

	For each platform the network service process is mentioned first
	as a reminder that it must start before the emulator.
	If either is stopped and restarted both must be

	The emulators, network interfacers, PC and client programs are

	Microsoft Windows
	_________________

	You can download RTA1 product to Windows and it contains some Windows executables
	for talking to RTA1 on a network and for constructing RTA1 target software,
	but not emulators or network service or client-side fponline

	These need a portation which is not currently resourced

	The largest problem is doing anything like Berkeley Packet Filter for network service.

	No packet filter product has been identified with sufficient documentation to use

	or with support for platform loopback interface which is indispensible

	Alternatives like raw socket types are have restrictions which preclude their use

	We have documented some guidance on installing openSUSE Linux as an alternative boot on a Windows platform
	but we accept no liability for mishaps leading to damage or loss.
	Follow instructions from the Linux supplier carefully

	The in-house RTA1-on-Linux server is a very old and cheap Windows PC + openSUSE Linux
	which has network wake-up selected in BIOS. It has been runnning emulated RTA1 for years

	The windows executables in the download are binary utilities and the following client-side programs

        i 72-bit fixed point calculator
        fp 96-bit floating calculator
        fp -U 192-bit floating calulator

	Browsers, telnet client and SNMP managers sourced from Microsoft and third parties
	also work with RTA1 servers
