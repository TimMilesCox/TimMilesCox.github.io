Embedded System Model

Application Core Map Overview

RTA1 system image development is oriented toward embedded servers 

	applications are ROMable shared instruction code and may be flashed or RAM loaded

        each process instance of each application has a copy of program data

        a new process unpacks its data sections into RAM from within the ROMable application image

	initial data load strings are modeled as a read-only file system

        executable space has a boundary between NVRAM at low order locations
	and RAM at higher addresses

	the last NVRAM objects before RAM are suites of fixed applications.
	These fixed applications initialise data into RAM and execute in NVRAM
