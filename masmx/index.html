masmx.7r3A [Build 5] binaries

Changes from 7r3 to 7r3A Build 5
masmx.7r3A revises and makes more consistent the assembly
of hierarchically structured and named code and data. Earlier
application code needs no changes

added -H option, allows unrounded floating constants

Changes in Build 5

displays errno if an input or an output file cannot be accessed pointing to cause of problem

option flag -I legalises inline code generation from $func macros

Changes in Build 4

Addition of :z tag to construct integer and floating constants seven storage words in size
Size tags exist already for [1|2|3|4|5|6||8|]-word number constants and they override
integer self-sizing and floating selectable default size
size-suffixes also position floating exponent field in label-equated values

Floating exponent underflow / overflow now marks the assembly in error
Exponent underflow / overflow in an assembled storage constant was previously remarked
but is now error-flagged

Earlier application code needs no changes

Introduced in masmx7r3A

Easy to Use Build Scripts for New or Updated masmx

addition of freestanding utility symbol
symbol extracts exported assembly labels
and makes them available as environmental variables
